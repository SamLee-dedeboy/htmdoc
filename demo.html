Links

While editing, clicking a link opens a small editor for both its text and its URL. Cmd/Ctrl+click opens the link in a new tab instead of editing it.

A normal link to a Google search, and a link styled as a button (to Wikipedia) — try a plain click (blocked), then Cmd/Ctrl+click (new tab).

Buttons and control labels

While editing, a plain click edits the label (a small input opens over it) and Alt/Option+click activates the control. Toggle Editing: OFF and everything behaves normally again.

Form state — the property-vs-attribute wall

What you type or pick in form controls lives in JavaScript properties, not in the markup — so it does not survive a save. The controls themselves, their labels, and fieldset/legend text are editable.

Images and figures

Click the image to replace it with one from your computer or edit its alt text; the picture's pixels themselves aren't editable. The figcaption is ordinary text — fully editable.

SVG graphics

Browsers don't support text editing inside SVG, so this tool adds its own path: click a label to edit it in a floating input. Shapes, paths, and colors are out of scope.

Media, canvas, and iframes — invisible to saving

These render things that aren't part of the document's markup. Video frames, audio, and canvas pixels can't be edited and aren't captured by a save; an iframe is a separate document the editor never enters.


This canvas is painted by a script on load. Save the page, open the copy: the drawing is gone from the markup — only the script that redraws it remains.

CSS-generated content

Text created by CSS ::before/::after isn't in the DOM, so it can't be clicked into or edited — only the real text after it can.

Scripts and script-generated content

Saving stores the rendered page next to the scripts that produced it, so content a script generates would duplicate on the next open — when a page is opened through the file browser, such content gets a red dashed outline (hover it for the reason).
